# DeepVision AI Risk Model

This folder adds the AI layer for the DeepVision attendance dashboard.

The model classifies each student as:

- `At Risk`
- `Not At Risk`

It uses attendance and academic features such as attendance percentage, total absences, longest absence streak, recent absence trend, required lectures, mid-sem marks, and internal marks.

## Important note about training labels

The fingerprint sensor records attendance. It does not directly know whether a student failed or was academically at risk.

For real supervised training, you need a label file like:

```csv
Enrollment,AtRisk
30,0
26,1
32,1
```

`AtRisk = 1` means the student was actually at risk based on prior semester result, teacher marking, low performance, or official academic criteria.

For a college prototype, you can use `--demo-labels`. That creates transparent rule-based demo labels from attendance and marks, then trains the PyTorch model on those labels.

## Install dependencies

```powershell
pip install -r ai_model/requirements.txt
```

## Train with sample data

```powershell
python ai_model/train.py --attendance ai_model/sample_attendance_iot.csv --labels ai_model/sample_labels.csv
```

Demo-label training:

```powershell
python ai_model/train.py --attendance ai_model/sample_attendance_iot.csv --demo-labels
```

The script writes:

- `ai_model/artifacts/student_risk_model.pt`
- `ai_model/artifacts/scaler.json`
- `ai_model/artifacts/metrics.json`
- `ai_model/artifacts/training_features.csv`

## Predict risk list for dashboard

```powershell
python ai_model/predict.py --attendance ai_model/sample_attendance_iot.csv
```

This writes:

```text
ai_model/artifacts/risk_predictions.json
```

The dashboard can read that JSON and display a teacher-facing alert list.

For your current frontend, keep the output path as:

```text
ai_model/artifacts/risk_predictions.json
```

`frontend/script.js` first tries to load this file. If the trained PyTorch prediction file is not present yet, the dashboard still shows live heuristic analysis based on the attendance data it already fetched from Google Sheets. That mode is useful for demos, but the real AI integration is the JSON generated by `predict.py`.

## Data flow

```text
Fingerprint sensor
    -> ESP32 sketch
    -> Google Apps Script
    -> Google Sheet attendance table
    -> ai_model/features.py
    -> ai_model/train.py or ai_model/predict.py
    -> risk_predictions.json
    -> DeepVision dashboard risk list
```

## Supported attendance input

The Python code accepts:

- CSV exports from Google Sheets
- JSON files shaped like the current Apps Script response:

```json
{
  "headers": ["Name", "Enrollment", "2026-04-01", "Required", "Can Skip", "MID SEM", "INTERNAL"],
  "data": [["Aryan", 30, "P", 2, 4, 34, 18]]
}
```

- Apps Script JSON URLs that return the same shape

## Dashboard integration idea

The dashboard now has a section called `AI Risk Analysis`.

Each item should show:

- student name
- enrollment number
- risk status
- confidence
- reason

Example:

```json
{
  "enrollment": 26,
  "name": "Samarth Saxena",
  "riskStatus": "At Risk",
  "confidence": 0.87,
  "reason": "low attendance, consecutive absences"
}
```

## How the full integration works

1. `array_trail_1.ino` sends fingerprint attendance to Google Apps Script.
2. Google Apps Script stores the attendance in Google Sheets.
3. `ai_model/features.py` reads the sheet export or Apps Script JSON and creates student-level ML features.
4. `ai_model/train.py` trains the PyTorch classifier using `AtRisk` labels.
5. `ai_model/predict.py` creates `risk_predictions.json`.
6. `frontend/script.js` loads `risk_predictions.json` and renders the result in the dashboard.

No Arduino code change is required for this AI dashboard step because the ESP32 is already doing its main job: collecting and sending attendance data.
